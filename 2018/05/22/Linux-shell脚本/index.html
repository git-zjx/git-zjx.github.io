反引号

反引号在 ESC 按键下面的按键上，和波浪号 ~ 同一按键

反引号允许将 shell 命令的输出赋值给变量

重定向输入输出

命令总是位于在命令行的开始位置，重定向符号指向数据流动的方向，小于号表示数据从输入文件流向命令，大于号表示命令产生的数据流向文件

两个大于号（>>）表示附加数据

管道

管道可以将一个命令的输出重定向到另一条命令，符号是竖条操作符（|）

Linux系统实际上同时运行两条命令，并在系统内部把它们连接在一起。第一条命令生成输出时，输出就立即发送给第二条命令。没有使用中间文件或者缓冲区传递数据

命令中可以使用数量不限的管道符，但行长度最多是255个字符

退出

echo $? 可以查看退出状态代码

退出状态代码

代码 描述
0 命令成功完成
1 通常的未知错误
2 误用 shell 命令
126 命令无法执行
127 没有找到命令
128 无效的退出参数
128+x 使用 Linux 信号 x 的致命错误
130 使用 Ctrl-C 终止的命令
255 规范外的退出状态

默认情况下，shell脚本以脚本中最后一条命令的退出状态退出

exit 命令可以指定退出状态，可以在 exit 中使用变量

退出状态码最高是 255，高于 255 会使用该状态码被 256 除了之后的余数作为状态码
