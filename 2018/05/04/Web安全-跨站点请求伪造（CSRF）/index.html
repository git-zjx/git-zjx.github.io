Web安全-跨站点请求伪造（CSRF）

2018-05-04
Web安全

  2. 成因
    1. 浏览器的Cookie策略
    2. P3P 头的副作用

    1. 验证码

  4. 参考

概念

通过伪装来自受信任用户的请求来利用受信任的网站

成因

浏览器的Cookie策略

浏览器所持有的Cookie分为两种：一种是「Session Cookie」，又称「临时Cookie」；另一种是「Third-party Cookie」，也成为「本地Cookie」

如果浏览器从一个域的页面中，要加载另一个域的资源，由于安全原因，某些浏览器会阻止 Third-party Cookie 的发送，但是 Session Cookie 会被发送

P3P 头的副作用

P3P Header 是 W3C 制定的一项关于隐私的标准，全称是「The Platform for Privacy Preferences」

如果网站返回给浏览器的 HTTP 头中包含有 P3P 头，则在某种程度上说，将允许浏览器发送「Third-party Cookie」

防御

验证码

验证码会强制用户必须与应用进行交互，能避免在用户不知情的情况下发送了请求

HTTP Referer 检测

检测请求是否来自合法的源，但服务器并非什么时候都能获取到 HTTP Referer ，可以用来检测 CSRF 的发生

CSRF Token

CSRF 能够成功的本质原因是重要操作的所有参数都可以被攻击者猜到，由于 Token 的存在，攻击者就无法再构建一个成功的请求
