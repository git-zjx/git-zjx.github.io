概念

通过把SQL命令插入到Web表单提交或输入域名或页面请求的查询字符串，最终达到欺骗服务器执行恶意的SQL命令

成因

用户输入的数据被SQL解释器执行

防御

使用预编译语句

防御 SQL 注入的最佳方式，使用预编译的SQL语句，SQL语句的语义不会发生改变，攻击者无法改变SQL结构，也就无法进行攻击

使用存储过程（原理类似预编译语句）

需要先将 SQL 语句定义在数据库中，尽量避免使用动态的 SQL
