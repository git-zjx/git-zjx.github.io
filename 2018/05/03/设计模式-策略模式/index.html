概念

定义一系列算法，将每一个算法封装起来，并让它们可以相互替换。策略模式让算法独立于使用它的客户而变化，也称为政策模式

组成

  • 抽象策略角色：策略类，通常由一个接口或者抽象类实现。
  • 具体策略角色：包装了相关的算法和行为。
  • 环境角色：持有一个策略类的引用，最终给客户端调用。

优点

  • 算法可以自由切换
  • 避免使用多重条件判断
  • 扩展性良好

缺点

  • 策略类数量增多
  • 所有策略类都需要对外暴露

使用场景

  • 多个类只有在算法或行为上稍有不同的场景
  • 算法需要自由切换场景
  • 需要屏蔽算法规则的场景
