处理信号

Linux 信号回顾

信号 值 描述
1 SIGHUP 挂起进程
2 SIGINT 中断进程
3 SIGQUIT 停止进程
9 SIGKILL 无条件终止进程
15 SIGTERM 如果可能的话终止进程
17 SIGSTOP 无条件停止，但不终止进程
18 SIGTSTP 停止或暂停进程，但不终止它
19 SIGCONT 重新启动停止的进程

默认情况下，bash shell 会忽略它接收的任何 SIGQUIT 和 SIGTERM 信号，以防止交互的 shell 意外终止

生成信号

中断进程

使用 Ctrl+C 组合键可以生成 SIGINT 信号，并将其发送给当前正在 shell 中运行的任意进程。不会在监视器上生成任何输出，只会停止当前在 shell 中运行的进程

暂停进程

使用 Ctrl+Z 组合键可以生成 SIGTSTP 信号，可以暂停任何在 shell 中运行的进程

方括号中的数字是 shell 分配的作业编号。shell 以作业形式引用 shell 中运行的每个进程，向每个作业分配唯一的作业编号

可以使用 ps 命令查看停止的作业，ps 命令将停止作业的状态显示为 T，表示该命令正被跟踪或者已经停止

如果想在停止作业仍处于活动状态时退出 shell，需要两次键入 exit 命令，shell 将退出，并终止停止的作业，也可以使用 kill 命令终止

终止作业时，最初不会得到任何响应，但在下一次使用某个生成 shell 提示符的操作时，会提示作业已终止

捕获信号

trap 命令可以指定能够通过 shell 脚本监控和拦截的 Linux 信号。如果脚本收到在 trap 命令中列出的信号，它将保护该信号不被 shell 处理，并在本地处理它

在 trap 命令行中，只需要列出希望 shell 执行的命令，以及希望捕获的信号列表（以空格分隔）。指定信号可以通过它们的数值或 Linux 信号名实现

捕获脚本退出

要捕获 shell 脚本退出，只需要向 trap 命令添加 EXIT 信号

当脚本到达常规退出点时，就会触发 trap，shell 将执行在 trap 命令行中指定的命令。使用 Ctrl+C 组合键退出也能捕获 EXIT

要移除捕获，使用破折号作为命令和想要恢复正常行为的信号列表

如果在移除捕获之前收到信号，脚本仍将根据 trap 命令处理该信号

以后台模式运行

将 & 符号放在命令之后，它将 bash shell 与命令相分离，并以独立的后台进程形式在系统上运行。

在不使用控制台的情况下运行脚本

nohup 命令运行另一个命令阻塞发送到进程的任何 SIGHUP 信号，可以防止在退出终端会话时退出进程

为了接受命令生成的任何输出，nohup 命令自动将 STDOUT 和 STDERR 消息重定向到称为 nohup.out 的文件

作业控制

查看作业

jobs 命令可以查看 shell 处理的当前作业

jobs 命令的一些选项

带有加号的作业被视为默认作业，带有减号的作业是在处理完当前默认作业之后将成为默认作业的作业。在某个时间点，只能有一个带加号的作业，也只能有一个带减号的作业

调度优先级

调度优先级是内核相对其他进程分配给某一进程的CPU时间量，默认情况下，从 shell 启动的所有进程在 Linux 系统上的调度优先级都相同

调度优先级是一个整数值，从 -20（最高优先级）到 +20（最低优先级）。默认为 0

nice 命令

nice 命令可以再启动命令时设置它的调度优先级

renice 命令

renice 命令可以修改指定运行进程的优先级

有如下限制：

  • 只能对拥有的进程使用 renice 命令
  • 只能使用 renice 命令将进程调至更低的优先级
  • 根用户可以使用 renice 命令将任何进程调至任何优先级
