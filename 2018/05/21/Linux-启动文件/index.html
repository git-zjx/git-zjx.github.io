登录 shell

当登录到 Linux 系统中时，bash shell 将作为登录 shell 启动。登录 shell 将查找 4 个不同的启动文件来处理其中的命令。处理文件的顺序如下：

/etc/profile 文件

bash shell 在系统上的主默认启动文件。系统上的每一个用户在登录时都将执行此启动文件。通常在该文件中设置全局变量。会迭代执行 /etc/profile.d 目录中的特定于应用程序的启动文件

$HOME 启动文件

这三个文件具有相同的功能，提供特定于用户的启动文件，用于定义特定于用户的环境变量。大多数 Linux 发行版只用其一

.bash_profile 启动文件首先将检查 HOME 目录中是否存在另一个名称为 .bashrc 的启动文件。如果存在，则启动文件将执行其中的命令。接下来，启动文件将添加一个目录到 PATH 环境变量中，目的是提供方在 HOME 目录中放置可执行文件的公共位置

交互式 shell

输入什么，解释什么就是交互式 shell，通常是登录 shell 派生出来的，会处理用户 HOME 目录中的 .bashrc 文件

.bashrc 文件

执行两个任务。首先，检查 /etc/bashrc 文件。其次，可以在 .bashrc 文件中定制个人命令别名或者私有脚本函数

/etc/bashrc 文件

设置一些非全局环境变量，也执行了/etc/profile.d 目录中的特定于应用程序的启动文件

非交互式 shell

用于系统执行shell脚本时调用

当 shell 开始一个非交互式 shell 进程时，它将检查 BASH_ENV 环境变量表示的待执行启动文件的名称。如果该变量有值，则 shell 将执行该文件中的命令
