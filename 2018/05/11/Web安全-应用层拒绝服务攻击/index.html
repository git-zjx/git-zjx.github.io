概念

利用合理的请求造成资源过载，导致服务不可用

攻击方式

  • 对一些消耗资源较大的应用页面不断发起正常请求
  • 构建不完整的 HTTP 请求占住 Web Server 连接不放
  • 发送 POST 包时，构造非常大的 Content-Length 值，然后以很低的速度发包，保持连接不断开
  • 客户端注入超长的 Cookie
  • ReDOS，基于正则实现

防御

限制请求频率

通过 IP 与 Cookie 定位一个客户端，如果客户端的请求在一定时间内过于频繁，则对之后来自该客户端的所有请求都重定向到一个页面
