Linux-显示数据

2018-05-28
Linux

了解输入和输出

标准文件描述符

文件描述符是一个非负整数，可以唯一标识会话中打开的文件。每个进程中最多可以有 9 个打开文件的描述符。bash shell 为特殊需要保留前 3 个文件描述符

文件描述符 缩写 描述
0 STDIN 标准输入
1 STDOUT 标准输出
2 STDERR 标准错误
STDIN

shell 通过 STDID 文件描述符从键盘接受输入，并在键入时处理每个字符

使用重定向符号（<）时，Linux 将使用重定向引用的文件替换标准的输入文件描述符

STDOUT

shell 的所有输出（包括 shell 中运行的程序和脚本）都定向到标准输出，也就是监视器

使用重定向符号（>）时，所有通常定向到监视器的输出都会重定向到 shell 指定的文件

STDERR

STDERR 处理错误信息，是 shell 发送错误消息的目的地

默认情况下，STDERR 与 STDOUT 指向相同的位置，也就是默认情况下，所有错误消息都显示在监视器上，但重定向 STDOUT 不会自动重定向 STDERR，错误消息仍会显示在监视器上

重定向错误

仅重定向错误

通过替换位于重定向符号正前方的文件描述符，可以仅重定向错误消息

使用这种方法，shell 仅重定向错误消息，不包括普通数据

重定向错误和数据

如果需要同时重定向错误和普通数据，则必须使用两个重定向符号。必须在希望重定向的数据前面方式相应的文件描述符，然后将它们指向相应的输出文件以保存数据

可以将 STDERR 和 STDOUT 输出重定向到同一个输出文件，使用 &> 符号即可

bash shell 自动使错误消息的优先级高于标准输出

在脚本中重定向输出

临时重定向

重定向到某个文件描述符时，必须在文件描述符编号前面添加 & 符号

永久重定向

使用 exec 命令通知 shell 在执行脚本期间重定向特定的文件描述符实现永久重定向

创建自己的重定向

创建输出文件描述符

exec 命令将文件描述符 3 重定向到一个文件，重定向到文件描述符 3 的 echo 语句会输出到该文件中，而不在监视器显示

重定向文件描述符

可以通过将标准文件描述符定向到一个备选的文件描述符，然后在将该文件描述符重定向会标准文件描述符来恢复标准文件描述符

首先，文件描述符 3 重定向到 STDOUT，表明任何发送到文件描述符 3 的输出都将显示在监视器上
然后，将 STDOUT 重定向到一个文件，表明任何发送到 STDOUT 的输出都将重定向到输出文件
最后，将 STDOUT 重定向到文件描述符 3，而文件描述符 3 仍然指向监视器，所以现在 STDOUT 指向了它原来的位置

STDIN 和 STDERR 与此类似

创建读取/写入文件描述符

可以使用同一个文件描述符从一个文件读取数据，同时向这个文件写入数据

读取文件和写入该文件时，shell 维护一个内部指针，指示文件内部的位置。读取和写入操作都发生在指针上次放置的位置

read 命令读取第一行数据后，文件指针指向第二行数据的第一个字符，当 echo 语句将数据输出到文件时，它将数据放在文件指针的当前位置，覆盖了此处的数据

关闭文件描述符

重定向到特殊符号 &-可以关闭文件描述符

关闭文件描述符后，如果稍后在脚本中打开同一个输出文件，shell 将使用新文件替换现有文件

禁止命令输出

可以将不希望显示或保存的输出重定向到 /dev/null 中，任何重定向到该位置的数据都将丢失且不会显示

使用临时文件

Linux 使用 /tmp 目录处理不需要永久保存的文件。大部分 Linux 发行版的系统配置都是在启动时自动删除 /tmp 目录中的任何文件

系统上任何账户都有权读取和写入 /tmp 目录中的文件

mktemp 命令可以轻松在 /tmp 文件夹中创建一个唯一的临时文件，shell 创建该文件但是不使用默认的 umask 值，仅向文件所有者分配读取和写入权限

创建本地临时文件

默认情况下，mktemp 在当前目录创建文件

mktemp 命令使用一个 6 字符代码替换 6 个 X，以确保文件名在目录中的唯一性，输出的是所创建的文件的名称

在 /temp 中创建临时文件

-t 选项强迫 mktemp 在系统的临时文件夹中创建文件。使用该特性时，mktemp 命令返回用于创建临时文件的完整路径名

创建临时目录

-d 选项让 mktemp 命令创建一个临时目录

记录消息

tee 命令就像管道的 T 型接头，它将 STDIN 的数据同时发送到两个目的地，一个是 STDOUT，另一个是 tee 命令指定的文件名

由于 tee 重定向来自 STDIN 的数据，因此可以与管道命令配置使用重定向任何命令的输出

默认情况下，tee 会覆盖输出文件，可以使用 -a 选项来避免

if-then

if 语句运行在 if 行定义的命令，如果命令的退出状态是 0，将执行 then 后面的所有命令。如果命令的退出状态是 0 以外的其他值，那么 then 后面的命令不执行

Linux-用户组

2018-05-21
Linux

/etc/group 文件

/etc/group 文件包含关于系统所使用的各用户组的信息，包含 4 个字段：

  • 用户组名称
  • 用户组密码
  • GID
  • 属于用户组的用户账户列表

用于系统账户的用户组将分配 500 以下的 GID，而用户的用户组将从 500 开始分配 GID

通过使用密码，用户组将允许非用户组成员临时成为用户组成员，不常用。

当用户使用某个用户组作为 /etc/passwd 文件里的默认用户组，用户账户不会作为成员出现在 /etc/group 中

创建新用户组

groupadd 命令用于创建用户组。创建新用户组时，系统默认不会分配任何用户。要添加新用户，需使用 usermod 命令

修改用户组

groupmod 用于修改用户组

/etc/passwd 文件

将登录名与相应的 UID 值进行匹配，包含以下信息：

  • 登录用户名
  • 用户密码
  • 用户账户的数值 UID
  • 用户账户的数值用户组 ID（GID）
  • 用户账户的文本描述（称为注释字段）
  • 用户 HOME 目录的位置
  • 用户的默认 shell

根用户账户是 Linux 系统的管理员，它的 UID 始终为 0。Linux 系统为各种非实际用户的函数创建了大量用户账户，称为系统账户。系统账户是一个特殊的账户，在系统中运行的服务使用它们访问系统资源

Linux-启动文件

2018-05-21
Linux

登录 shell

当登录到 Linux 系统中时，bash shell 将作为登录 shell 启动。登录 shell 将查找 4 个不同的启动文件来处理其中的命令。处理文件的顺序如下：

/etc/profile 文件

bash shell 在系统上的主默认启动文件。系统上的每一个用户在登录时都将执行此启动文件。通常在该文件中设置全局变量。会迭代执行 /etc/profile.d 目录中的特定于应用程序的启动文件

$HOME 启动文件

这三个文件具有相同的功能，提供特定于用户的启动文件，用于定义特定于用户的环境变量。大多数 Linux 发行版只用其一

.bash_profile 启动文件首先将检查 HOME 目录中是否存在另一个名称为 .bashrc 的启动文件。如果存在，则启动文件将执行其中的命令。接下来，启动文件将添加一个目录到 PATH 环境变量中，目的是提供方在 HOME 目录中放置可执行文件的公共位置
